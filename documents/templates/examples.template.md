# Examples Template

**목적:** 실제 문제를 해결하는 방법을 보여줌.  
**대상 위치:** `EXAMPLES.md` 또는 `docs/examples/`

---

## 항목 템플릿 (예제당)

```markdown
## 예제: [Problem Statement]

**문제 파악:** 
[scenario] 상황에서 [accomplish goal]을(를) 달성해야 합니다.

**해결책:**

\`\`\`[language]
[complete, working solution code]
\`\`\`

**출력:**
\`\`\`
[expected output of running the code]
\`\`\`

**설명:**

- **Line X-Y**: [이 부분이 수행하는 작업과 이유]
- **핵심 포인트**: [중요한 설계 결정 또는 주의 사항]
- **일반적인 변형**: [유사한 목표 X를 위해 코드를 수정하는 방법]

**관련 예제:**
- [Example 2] — 에러 처리 추가 방법
- [Example 5] — 여러 항목으로 이 패턴 확장하기

---
```

---

## 작성 가이드라인

1. 복잡도 순으로 그룹화 (기본 → 중급 → 고급)
2. 사용 사례별 그룹화 (데이터 처리, API 통합 등)
3. 각 예제는 독립적이어야 함 (단독으로 실행 가능)
4. 다음 사항을 최소한 하나씩 포함:
   - 오류 처리 예제 한 개
   - 성능/확장 예제 한 개
   - "자주 묻는 실수" 예제 한 개
5. 실제 출력을 포함 (실제 콘솔 출력을 복사-붙여넣기)
6. 각 예제마다 핵심 학습 포인트를 강조
